OBS Studio Setup Guide

Complete guide to set up OBS Studio for streaming to Interviews.tv RTMP server

RTMP Connection Details
RTMP Server URL:
rtmp://localhost:1935/live
Stream Key:
test-stream-key

Step 1: Install OBS Studio

1Download OBS Studio

Choose your installation method:

Option A: Package Manager (Recommended)
sudo apt update && sudo apt install obs-studio
Option B: AppImage (Already Downloaded)
chmod +x obs-studio.AppImage
./obs-studio.AppImage
Option C: Official Website
Download from: obsproject.com

OBS Studio Download Page

Step 2: Configure OBS for RTMP Streaming

2Open OBS Studio Settings

Launch OBS Studio and follow these steps:

  1. Click File → Settings
  2. Navigate to Stream in the left sidebar
  3. Set Service to "Custom..."
  4. Enter the RTMP server details

OBS Settings → Stream
3RTMP Stream Configuration
Service:
Custom...
Server:
rtmp://localhost:1935/live
Stream Key:
test-stream-key

Step 3: Set Up Your Scene

4Add Video Sources

Common sources for interview streaming:

  • Video Capture Device - Webcam or camera
  • Display Capture - Screen sharing
  • Window Capture - Specific application
  • Audio Input Capture - Microphone
  • Audio Output Capture - System audio
Quick Setup:
1. Right-click in Sources
2. Add → Video Capture Device
3. Select your webcam
4. Add → Audio Input Capture
5. Select your microphone

Add Sources Menu

Step 4: Start Streaming

5Begin Your Stream
  1. Click "Start Streaming" button
  2. Check the RTMP Test Interface
  3. Verify your stream appears in the active streams list
  4. Monitor the HLS output URL
Success Indicators:
✅ Green "LIVE" indicator in OBS
✅ Stream appears in test interface
✅ HLS playlist generates
✅ No connection errors

Start Streaming Button

Troubleshooting

Common Issues
  • Connection Failed: Check RTMP server is running
  • No Video: Verify camera permissions
  • No Audio: Check microphone settings
  • Poor Quality: Adjust bitrate settings
Quick Fixes
# Check RTMP server status
curl http://localhost:8081/health

# Test RTMP connection
telnet localhost 1935
